## 3. 代码提交规范
### 提交comment格式
    提交类型: 内容

    例: 
    feat: 登录功能
    fix: 修复登录bug 

### 提交类型
    feat - 新功能feature
    fix - 修复bug
    docs - 仅仅修改文档，如README，CHANGELOG等
    style - 修改代码格式，样式等，不改变逻辑
    refactor - 重构、优化(既不增加新功能，也不是修复bug)
    perf - 性能优化
    test - 测试用例，包括单元测试、集成测试等
    chore - 改变构建流程、增加依赖库、工具等
    revert - 回退
    build - 构建
